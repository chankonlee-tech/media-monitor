# 언론계 인사/부고 모니터링 시스템

연합뉴스 인사/부고 페이지에서 **언론사 관련 소식만 자동으로 수집**하여 MS Teams로 알림을 보내는 시스템입니다.

## 🎯 주요 기능

1. **자동 스크래핑**: 연합뉴스 인사/부고 페이지에서 오늘 날짜의 게시물 수집
2. **AI 필터링**: Gemini AI를 사용하여 언론사/방송사/미디어 관련 소식만 선별
3. **맞춤형 리스트**: 회사 관리 언론사 리스트(25,903개)를 기반으로 정확한 필터링
4. **기사 본문 수집**: 각 기사의 제목과 본문 내용을 자동으로 스크래핑
5. **중복 방지**: GitHub Artifact를 활용하여 이미 발송된 소식은 재전송하지 않음
6. **MS Teams 알림**: Adaptive Cards v1.5 형식으로 카드 뉴스 형태의 아름다운 알림 전송
7. **자동화**: GitHub Actions를 통한 하루 5회 자동 실행

## ✨ Adaptive Cards v1.5 디자인

각 기사가 카드 뉴스 형태로 깔끔하게 표시됩니다:

- **헤더**: 📰 언론계 인사/부고 소식 + 날짜/시간
- **섹션별 구분**: 
  - 📰 인사 (파란색 강조)
  - 🖤 부고 (회색 톤)
- **각 기사 카드**:
  - 제목 (굵게, 강조색)
  - 본문 내용 (전체 표시)
  - "기사 전체보기" 버튼
- **반응형 디자인**: 모바일/데스크톱 모두 최적화

## 📋 언론사 리스트 필터링

### 개선된 필터링 시스템 (v2.0)

이 시스템은 **업데이트된 미디어리스트**를 기반으로 **100% 정확도**의 필터링을 제공합니다:

- **원본 데이터**: 28,089줄 (다양한 언론사 정보 포함)
- **정제 후**: 25,903개 고유 언론사 (중복 및 잘못된 항목 제거)
- **정규화**: 25,685개 핵심 언론사명 (괄호 제거 등)
- **필터링 정확도**: 100% (테스트 케이스 30/30 통과)

#### 포함된 주요 언론사

- **신문사**: 조선일보, 중앙일보, 동아일보, 한국경제, 매일경제 등
- **방송사**: KBS, MBC, SBS, JTBC, YTN, 연합뉴스TV 등
- **통신사**: 연합뉴스, 뉴시스, 뉴스1 등
- **온라인**: 아주경제, 이데일리, 뉴스핀, 비즈워치, 트루스데일리 등
- **전문지**: 화장품신문, 뷰티경제, 뷰티한국, 헬스경향 등
- **미디어 플랫폼**: 네이버(미디어 부문), 카카오(콘텐츠 부문) 등

### 정교화된 필터링 방식

#### 1단계: 미디어리스트 정제
```bash
python clean_media_list.py
```
- 잘못된 형식 제거 (HTML 엔티티, 숫자로 시작 등)
- 중복 제거 및 정규화
- 출력: `media_list_cleaned.txt`

#### 2단계: AI 기반 필터링
- **정규화된 언론사명 활용**: 괄호, 공백 등 무시하고 핵심 언론사명 매칭
- **부분 매칭 지원**: 
  - "조선일보 논설위원" → 조선일보 매칭 ✓
  - "MBC 보도국장" → MBC 매칭 ✓
  - "세계일보(디지털미디어국)" → 세계일보 매칭 ✓
- **미디어 직책 탐지**:
  - "네이버 미디어 본부장" ✓
  - "카카오 콘텐츠 대표" ✓
- **업계 전문지 포함**: 화장품신문, 뷰티한국, 트루스데일리 등

#### 3단계: 중복 방지 및 전송
- GitHub Artifact 기반 발송 내역 관리
- 새로운 항목만 MS Teams로 전송

## 🤖 GitHub Actions 자동화

### 설정 방법

1. GitHub Repository Settings → Secrets and variables → Actions
2. 다음 시크릿 추가:
   - `TEAMS_WEBHOOK_URL`: MS Teams Webhook URL
   - `GEMINI_API_KEY`: Gemini API 키

### 실행 스케줄

- **하루 5회 자동 실행** (한국시간 기준):
  - 오전 10시 (UTC 01시)
  - 오후 12시 (UTC 03시)
  - 오후 2시 (UTC 05시)
  - 오후 4시 (UTC 07시)
  - 오후 6시 (UTC 09시)
- **수동 실행**: GitHub Actions 탭에서 "Run workflow" 버튼으로 즉시 실행

⚠️ **중요**: GitHub Actions의 scheduled 워크플로우는 정확한 시간을 보장하지 않습니다.
- 무료/public 저장소는 우선순위가 낮아 **10분~1시간 정도 지연**될 수 있습니다.
- 예상 시간에 알림이 오지 않더라도 정상 작동 중일 수 있습니다.
- 더 정확한 시간이 필요한 경우 수동 실행을 권장합니다.

### 중복 발송 방지

- **GitHub Artifact 활용**: 각 실행 시 발송한 항목을 `sent_items.json` 파일로 저장
- **자동 비교**: 다음 실행 시 이전 발송 내역과 비교하여 새로운 항목만 전송
- **보관 기간**: 90일간 발송 내역 유지 (GitHub 기본 설정)

## 🔍 트러블슈팅

### GitHub Actions 스케줄 지연

**증상**: 예정된 시간(10시, 12시, 14시 등)에 알림이 오지 않음

**원인**: 
- GitHub Actions의 scheduled 워크플로우는 정확한 시간을 보장하지 않습니다
- 무료/public 저장소는 우선순위가 낮아 큐에서 대기할 수 있습니다
- 피크 시간대에는 지연이 더 심할 수 있습니다

**해결 방법**:
1. **대기**: 보통 10분~1시간 내에 실행됩니다
2. **수동 실행**: GitHub Actions 탭 → "Run workflow" 버튼
3. **확인**: Actions 탭에서 워크플로우 실행 상태 및 로그 확인

### Artifact 에러

**증상**: `Unable to download artifact(s): Artifact not found for name: sent-items`

**원인**:
- 첫 실행이거나 이전 artifact가 만료됨 (90일 보관)
- 워크플로우 설정 변경으로 artifact가 초기화됨

**해결 방법**:
- 정상적인 동작입니다 (continue-on-error 설정으로 계속 진행)
- 다음 실행부터는 artifact가 정상적으로 생성되어 중복 방지가 작동합니다

### 언론사가 필터링되지 않는 경우

1. **리스트 확인**: `python test_media_list.py`로 해당 언론사가 포함되어 있는지 확인
2. **리스트 추가**: `media_list.txt`에 언론사명 추가
3. **이름 확인**: 제목의 언론사명과 리스트의 이름이 정확히 일치하는지 확인
   - 예: "코리아타임스" vs "코리아타임즈" (스 vs 즈)

### API 오류

- **GEMINI_API_KEY 없음**: `.env` 파일에 올바른 API 키 설정 확인
- **API 할당량 초과**: Gemini API 할당량 확인
- **네트워크 오류**: 인터넷 연결 상태 확인

### Teams 알림 실패

- **TEAMS_WEBHOOK_URL 없음**: `.env` 파일에 올바른 Webhook URL 설정 확인
- **Webhook 만료**: MS Teams에서 새 Webhook URL 생성

### 중복 발송 관련

- **중복 항목이 계속 발송됨**: `sent_items.json` 파일이 제대로 저장/로드되는지 확인
- **모든 항목이 중복으로 필터링됨**: GitHub Actions에서 artifact가 정상적으로 업로드/다운로드되는지 확인
- **artifact 없음**: 첫 실행 시에는 정상이며, 이후 실행부터 중복 방지가 작동함
