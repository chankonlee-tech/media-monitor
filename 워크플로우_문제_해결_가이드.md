# GitHub Actions 자동 실행 문제 해결 가이드

## 📋 변경 사항

### 1. Repository Dispatch 트리거 제거
**이유**: Repository Dispatch는 외부 API 호출이 필요한 트리거로, 자동으로 실행되지 않습니다.

### 2. Cron 스케줄 단순화
```yaml
이전: cron: '0 23,0-9 * * *'  # 너무 복잡한 스케줄
변경: cron: '0 0-9 * * *'  # 1시간 간격 10회 실행
```

**실행 시간 (한국시간 기준)**:
- 오전 09시 (UTC 0시)
- 오전 10시 (UTC 1시)
- 오전 11시 (UTC 2시)
- 오후 12시 (UTC 3시)
- 오후 01시 (UTC 4시)
- 오후 02시 (UTC 5시)
- 오후 03시 (UTC 6시)
- 오후 04시 (UTC 7시)
- 오후 05시 (UTC 8시)
- 오후 06시 (UTC 9시)

### 3. 실행 기록 개선
`last_run.txt` 파일에 다음 정보가 기록됩니다:
- 한국시간 (KST)
- UTC 시간
- 워크플로우 트리거 방식 (schedule/workflow_dispatch)
- 실행 번호

## 🔍 실행 여부 확인 방법

### GitHub 웹사이트에서 확인
1. https://github.com/chankonlee-tech/media-monitor 접속
2. **"Actions"** 탭 클릭
3. 왼쪽에서 **"언론계 인사/부고 모니터링"** 워크플로우 선택
4. 실행 기록 확인:
   - ✅ 초록색: 성공
   - ❌ 빨간색: 실패
   - 🟡 노란색: 진행 중
   - ⚪ 회색: 건너뜀/취소

### 로컬에서 확인
```bash
# 저장소를 최신 상태로 업데이트
git pull origin main

# last_run.txt 파일 확인
cat last_run.txt
```

## 🚨 문제 해결

### 스케줄이 실행되지 않는 경우

#### 원인 1: GitHub Actions 비활성화
**확인 방법**:
1. Repository → Settings → Actions → General
2. "Actions permissions" 확인

**해결 방법**:
- "Allow all actions and reusable workflows" 선택
- "Workflow permissions"를 "Read and write permissions"로 설정
- Save 클릭

#### 원인 2: 60일간 저장소 활동 없음
**현상**: 저장소에 60일간 커밋/활동이 없으면 스케줄 워크플로우가 자동으로 비활성화됩니다.

**해결 방법**:
- 워크플로우가 실행될 때마다 `last_run.txt`를 자동 커밋하도록 설정되어 있어 이 문제는 발생하지 않습니다.

#### 원인 3: Secrets 미설정
**확인 방법**:
1. Repository → Settings → Secrets and variables → Actions
2. 다음 Secrets가 있는지 확인:
   - `GEMINI_API_KEY`
   - `TEAMS_WEBHOOK_URL`

**해결 방법**: 없으면 추가

#### 원인 4: GitHub Actions 지연
**현상**: 무료 플랜에서는 스케줄이 5~60분 정도 지연될 수 있습니다.

**해결 방법**:
- 특별히 할 것 없음 (정상 동작)
- 정확한 시간에 실행이 필요하면 유료 플랜 사용

### 수동으로 즉시 실행하기
1. GitHub Actions 탭 이동
2. "언론계 인사/부고 모니터링" 워크플로우 선택
3. 오른쪽 상단 **"Run workflow"** 버튼 클릭
4. "Run workflow" 다시 클릭

## 📊 실행 스케줄

매일 한국시간 기준:
- 오전 9시부터 오후 6시까지
- 1시간 간격으로 총 10회 실행

## 🔔 알림 수신 여부

### 정상 작동 시
- MS Teams에 알림이 옴
- 새 소식이 없어도 "현재까지 추가된 인사/부고가 없습니다" 메시지가 표시됨

### 작동하지 않는 경우
- Teams에 아무 알림도 오지 않음
- GitHub Actions에서 워크플로우 실행 기록 확인 필요

## 💡 추가 개선 사항

### 더 자주 실행하고 싶다면
`.github/workflows/run.yml` 파일의 cron 스케줄 수정:

```yaml
# 2시간마다 실행 (오전 8시~오후 8시)
- cron: '0 23,1,3,5,7,9,11 * * *'

# 1시간마다 실행 (오전 8시~오후 8시)
- cron: '0 23,0-12 * * *'
```

### Repository Dispatch를 사용하고 싶다면
외부 시스템에서 다음 API를 호출하면 됩니다:

```bash
curl -X POST \
  -H "Accept: application/vnd.github.v3+json" \
  -H "Authorization: token YOUR_GITHUB_TOKEN" \
  https://api.github.com/repos/chankonlee-tech/media-monitor/dispatches \
  -d '{"event_type":"trigger-monitoring"}'
```

단, 이는 수동 트리거이며 자동으로 실행되지 않습니다.
